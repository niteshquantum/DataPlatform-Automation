# 24 — Glossary

| Term | Meaning (as built) |
|------|----------------------|
| **Jenkinsfile** | Main declarative pipeline (`jenkins/Jenkinsfile`); parameterized, `agent none`, routes to Windows/Ubuntu agents. |
| **Standalone pipeline** | Per-DB/OS `.groovy` job (`jenkins/<db>/<os>/<phase>_pipeline.groovy`). |
| **Direct local** | Running `scripts/batch/*.bat` or `scripts/bash/*.sh` without Jenkins. |
| **Shared core** | The batch/bash/PowerShell/Python scripts that all three entry modes call. |
| **Agent label** | Jenkins node selector; `windows-node` / `ubuntu-node` used by the main pipeline. |
| **Terraform null_resource** | Terraform resource with `local-exec` that runs a local install/start script. |
| **local-exec** | Terraform provisioner that executes a command on the machine running Terraform. |
| **Liquibase** | Tool that applies versioned schema/object changelogs (`master.xml`, `master_objects.xml`). |
| **Changelog** | Liquibase XML describing schema/object changes; applied via `liquibase update`. |
| **master.xml** | Schema changelog entry point (runtime-generated includes). |
| **master_objects.xml** | Objects changelog, generated by `generate_master_objects.py`. |
| **DATABASECHANGELOG / LOCK** | Liquibase tracking tables ensuring idempotent, locked applies. |
| **data_loader.py** | Python loader for CSV/JSON into the DB; idempotent via SHA256 history. |
| **incoming/** | Expected runtime input directory for dataset files (not in repo). |
| **archive/ / failed/** | Directories the loader moves processed / failed files into. |
| **bootstrap_generator.py** | Generates object SQL + XML + master_objects.xml (generation only). |
| **deploy_objects.py** | Deploys `master_objects.xml` via the DB/OS Liquibase runner. |
| **ObjectDetector** | Reads schema metadata to drive object generation. |
| **database_objects.json** | Capability matrix: which object types each DB supports. |
| **RBAC** | Role-based access control; file-based auth + authorization gate. |
| **permissions** | Keys like `postgresql.setup`; checked against a role's granted list. |
| **roles.json** | Maps roles (Admin/Developer/QA/Viewer) to permission lists. |
| **credentials.json** | Users with SHA256-hashed passwords and assigned roles. |
| **logger.py** | Writes `logs/<db>/<action>/build_N/execution.json` stage tracking. |
| **generate_report.py** | Renders `report.html` + `report.json` from the execution log. |
| **metadata/** | Machine-written store: profiling, discovery, assessment, reconciliation. |
| **PROJECT_ROOT** | Repo root, computed by `set_project_root.bat/.sh`. |
| **config_loader.py** | Loads `config/<os>/<db>.conf` based on `platform.system()`. |
| **CLEANUP_MODE** | `PRESERVE_DATA` / `DELETE_DATA`, passed into cleanup pipelines. |
| **LOAD_MODE** | `skip` (default) / `force` / `reload` (truncate) for the data loader. |
| **DISCLAIMER** | Docs derived from static code inspection; no runtime execution was performed. |
